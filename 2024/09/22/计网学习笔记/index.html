计网学习笔记

发表于2024-09-22|更新于2025-02-17|lesson note

4.接入网与网络媒体

接入网：将端系统物理连接到其边缘路由器的网络

边缘路由器：从端系统到任何其他远程端系统路径上的第一台路由器

1.网络接入

端系统怎样连接到边缘路由器？

1.家庭接入

DSL、电缆、FTTH、拨号、卫星

2.企业接入

局域网(LAN)------>利用以太网技术

无线局域网(无线以太网，wireless LAN，WIFI)

以太网技术：

  • 共享以太网： 端系统共享以太网的传输速率；

  • 交换以太网： 多个用户可同时使用全部带宽通信。

  • 以太网的MAC协议提供的是无连接服务

3.无线接入

无线局域网(无线以太网，wireless LAN，WIFI)

广域无线接入网

2.物理媒体

将网络中不同端系统互相连接起来的物理线路，是进行数据传输的物理通路

分类

  • 导引型媒体
    • 电波沿着固体每题传播
    • 双绞线、同轴电缆、光缆
  • 非导引型媒体
    • 电波在空气/外层空间传播
    • 无线电

5.ISP互联

端系统经过一个接入网与因特网相连

因特网边缘的接入网络通过分层的ISP与因特网其他部分相连

6.时延

1.分组传输过程

2.时延类型

  • 节点处理时延：决定分组去处、校验分组完整性

    • 一般在微秒级或更低的数量级
    • 表示为：dprocd_{proc}dproc​

  • 排队时延：在交换机中等待推送，是以分组为单位的，缓存中如果该分组前面没有分组等待，那么整个分组都没有排队时延

    • 一般在毫秒或微秒级
    • 表示为：dqueued_{queue}dqueue​

  • 传输时延：在交换机的输出端口等待推入链路的时延

    • 传输时延 = L/RL/RL/R，分组长度=L
      bit，链路传输速率=R bps
    • 一般在毫秒或微秒级
    • 表示为：dtransd_{trans}dtrans​

  • 传播时延：在链路两节点间的时延

    • 传播时延=d/sd/sd/s，两节点间距离=d，链路传播速率=s m/s
    • 通常s在2⋅1082\cdot 10^82⋅108到3⋅1083\cdot10^83⋅108之间，取决于物理媒体性质
    • 一般在毫秒级
    • 表示为dpropd_{prop}dprop​

3.节点总时延

节点总时延表示为：dnodald_{nodal}dnodal​

$$d_{nodal} = d_{proc} + d_{queue} + d_{trans} + d_{prop}$$

4.排队时延

  • 不同分组，路径(route)不同，排队时延不同
  • 使用统计量测度
  • 排队时延大小的度量：流量强度
    • 流量强度 = 比特到达队列的速率与比特从队列中推出的速率比

设R= 链路速率 (bps)，L= 分组长度 (比特)，a=平均分组到达速率(每秒到达了几个分组，pkt/s，也就是packet/s)

则流量强度

I=La/RI = La/RI=La/R(比特到达队列速率=LaLaLabps，即每秒到达的比特的数量)

La/R>1La/R>1La/R>1时，分组到达速率大于推出速率，最终平均时延无穷大

故，设计系统时，流量强度不能大于1

5.分组丢失

当到达的分组发现队列已满，没有空间存储，被丢弃

  • 丢弃的分组可能由前面的节点或源端系统重传(面向连接服务)，或根本不重传(无连接服务)

6.端到端时延

从源到目的地的时延

如果忽略排队时延，那么：

路由器和源主机的处理时延是dprocd_{proc}dproc​

路由器和源主机的输出速率是Rbit/sR bit/sRbit/s

每条链路的传播时延是dpropd_{prop}dprop​

每个节点时延：dproc+dtrans+dpropd_{proc} + d_{trans} + d_{prop}dproc​+dtrans​+dprop​

端到端时延：dend−end=N(dproc+dtrans+dprop)d_{end-end} = N(d_{proc} + d_{trans} + d_{prop})dend−end​=N(dproc​+dtrans​+dprop​)

dtrans=L/Rd_{trans} = L/Rdtrans​=L/R

需要注意一点就是，如果题目中没有提到源主机处理时延，就不要考虑，仅考虑路由器/交换机的处理时延，也就是(N−1)dproc(N-1)d_{proc}(N−1)dproc​

7.协议层次与服务模型

1.分层体系结构

为了研究和设计方便，一般采用分层的方法按照功能划分为若干个层次

分层特点

  • 每层功能独立
  • 每两个相邻层之间有一个逻辑接口，可交换信息
    • 一般我们称这个逻辑接口为：服务访问点SAP
  • 上一层建立在下一层基础上，上一层可调用下一层的服务，下一层为上一层提供服务

分层优点

  • 使复杂系统简化
  • 易于维护、系统的更新
    • 某层功能的变化，不会影响系统其余部分

2.协议分层

协议：控制网络中信息的发送和接收

定义了通信实体之间交换报文的格式和次序，以及在报文传输和/或接收或其他事件所采取的动作

协议分层：
采用**分层(layer)**的方式组织协议及实现协议的网络硬件和软件

1.说明

  • 每层都有相应的一系列协议，如HTTP，TCP
  • 每层协议通过软件、硬件或两者结合实现
  • 每层协议可分布在网络的不同组件中：如端系统、分组交换机
    • 如应用层协议在端系统中用软件实现
  • 协议栈：各层所有协议的集合

2.服务模型

上层调用下层的服务，下层为上层提供服务

  • 示例：
    • 第n层提供报文的可靠传输------依赖于第n-1层的不可靠报文传输+本层的检测和重传丢失报文的功能
  • 缺点：
    • 某些功能可能在不同层重复出现
    • 某层的功能可能需要仅存在于其他层的信息

3.分层后数据传递

主机间数据传送实际上并不是在对等层间直接进行,而是通过相邻层间的传递合作完成

4.因特网协议栈

因特网协议栈分为5层(从低到高计数)：

  • 应用层(第5层)：
    • 功能：提供各种网络应用
    • 传输单位：报文
    • 协议：FTP，SMTP，HTTP等
  • 运输层(第4层)：
    • 功能：在应用程序的客户机和服务器之间提供传输应用层报文服务
    • 传输单位：报文段
      • 更具体点说，TCP协议的数据传输单位是报文段，UDP协议的数据传输单位是数据报
    • 协议：TCP，UDP等
  • 网络层(第3层)：
    • 功能：主机和主机之间传输网络层分组
    • 传输单位：数据报
    • 协议：IP协议，选路协议等
  • 链路层(第2层)：
    • 功能：在邻近单元之间传输数据
    • 传输单位：帧
    • 协议：PPP，以太网等
  • 物理层(第1层)：
    • 功能：在节点之间传输比特流
    • 传输单位：比特
    • 协议：传输媒体等

5.OSI模型

协议栈不止一个，OSI模型就是不同于因特网协议栈的另一个协议栈

  • **表示层：**通信用户之间数据格式的转换、数据压缩以及加解密等
  • **会话层：**对数据传输进行管理，包括数据交换的定界、同步，建立检查点等

还记得网络边缘和网络核心吗？协议栈与之也是有映射关系的。

因特网协议栈中，应用层、运输层是网络边缘(资源子网)，网络层、链路层、物理层是网络核心(通信子网)

OSI协议栈中，应用层、表示层、会话层、运输层是网络边缘，网络层、数据链路层、物理层是网络核心

6.节点设备分层

与端系统类似，路由器和链路层交换机以分层方式组织网络硬件和软件，通常只实现低几层

路由器：实现1-3层(物理层、链路层、网络层)，能够实现IP协议

链路层交换机：实现1-2层(物理层、链路层)，能识别第二层地址，如以太网地址

主机：实现所有5个层次

7.报文传递

源主机：由高层向低层逐层传递(封装)

  • 每层传递的数据分为：首部字段和有效载荷字段
    • 首部字段：每层自己的首部信息HiH_{i}Hi​
    • 有效载荷字段：相邻上层传下来的数据

目的主机：由低层向高层逐层传递(解封)

  • 物理层接收，并沿协议栈逐层向上传递，每层去除对应的首部，恢复原报文
  • 如果报文很长，传输时，可先分成多个报文段，每隔报文段在网络层再分为多个数据报
